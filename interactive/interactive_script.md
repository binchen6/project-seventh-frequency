**为什么不选Ren'Py？**
- Ren'Py需要Python运行时，Web部署需转译（RenPyWeb），体积大、加载慢
- 纯HTML5方案：单HTML文件即可运行，GitHub Pages直接部署，零门槛
- Web Audio API原生支持频率可视化（正弦波、频谱分析），完美契合「第七频率」主题

## 1.3 「Art Deco故障风」视觉概念

**设计灵感来源**：
- 上海1930年代Art Deco建筑（国际饭店、百乐门）的几何线条与对称美学
- 科幻「故障艺术」（Glitch Art）的数字撕裂感
- 民国月份牌招贴画的饱和色彩与装饰纹样

**核心视觉元素**：
```
常规场景：         频率异常场景：        记忆传输场景：
┌──────────┐      ┌──────────┐       ┌──────────┐
│  sepia   │  →   │  sepia   │   →   │  黑白+   │
│  黄褐色  │      │  + 蓝紫  │       │  金色边  │
│  老照片  │      │  glitch  │       │  缘闪烁  │
│  质感   │      │  条纹    │       │         │
└──────────┘      └──────────┘       └──────────┘
```

**具体实现**：
- **CSS变量系统**：定义 `--sepia-base`, `--neon-cyan`, `--art-deco-gold` 等主题色
- **SVG装饰边框**：Art Deco几何纹样作为场景边框和UI元素
- **Glitch效果**：CSS `clip-path` + `transform: skew()` 模拟数字信号干扰
- **字体组合**：标题用「站酷庆科黄油体」（复古装饰感），正文用「思源宋体」（可读性）

### 微互动（对话中选择回应方式）

微互动不改变路线，但影响角色好感度和后续对话内容。

**示例**：第二章场景3，林若兰第一次见沈默

```
林若兰：「沈先生这个问题很有意思。从理论上讲，特定频率的超声波可以影响神经系统，但『假死』？那是科幻小说的情节。」

┌─────────────────────────────────────────────────┐
│  沈默的回应（微互动，3秒窗口）                    │
│  ┌─────────────┐ ┌─────────────┐ ┌────────────┐│
│  │ A. 沉默观察 │ │ B. 追问疤痕 │ │ C. 直接质疑││
│  │  DRAMA+5   │ │  CLUE+1    │ │  INTEL+5   ││
│  │  林若兰+5  │ │  林若兰+0  │ │  林若兰-5  ││
│  └─────────────┘ └─────────────┘ └────────────┘│
└─────────────────────────────────────────────────┘
```

- **A. 沉默观察**：沈默不说话，只盯着林若兰的手。她注意到他的目光，下意识地藏了藏手。对话继续，但氛围微妙改变。
- **B. 追问疤痕**：触发隐藏线索CL-03的前置条件。林若兰的回答含糊，但眼神闪烁。
- **C. 直接质疑**：「林小姐，您手背的疤痕……是怎么来的？」林若兰表面微笑，但好感度-5。后续她会更保留。

### 解谜环节（利用「民国科技」主题）

**解谜1：频率匹配（第二章）**

陈子轩解码唱片后，屏幕上出现一系列波形。玩家需要拖动滑块，将频率调整到7830 Hz。

```
┌──────────────────────────────────────────┐
│  频率： ████████████████████░░░░░░  │
│         6000    7000    7830    9000   │
│                                          │
│  [◄────滑块────►]                        │
│                                          │
│  波形显示：∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿  │
│                                          │
│  提示：「密纹的间距……像某种编码」       │
└──────────────────────────────────────────┘
```

- 滑块接近7830时，波形开始稳定，BGM中的杂音减少
- 偏离时，波形紊乱，屏幕出现glitch效果
- **成功奖励**：TECH+15，陈子轩好感+10
- **失败惩罚**：可重试3次，第3次失败后陈子轩直接给出答案，但TECH只+5

**解谜2：摩尔斯电码（第三章）**

沈默在地下实验室发现无线电发报器，需要输入正确的摩尔斯电码来发送求救信号。

- 屏幕显示「SOS」的摩尔斯电码提示：「••• ─── •••」
- 玩家点击「滴」和「答」按钮，输入正确序列
- **成功**：触发林若兰救援，跳过被捕剧情
- **失败**：触发DE-01死亡结局（被捕）

**解谜3：声波拼图（第五章）**

最终抉择前，沈默需要「剥离」陶启明的数据。玩家需要在屏幕上拼接声波片段，形成「反相波」。

- 屏幕显示陶启明的数据波形（红色）和若干碎片波形（灰色）
- 玩家拖动碎片，使它们与红色波形形成镜像（相位反转）
- **成功**：解锁「选择性剥离」方案
- **失败**：只能选择「彻底摧毁」方案

# 三、「回响系统」—— 蝴蝶效应可视化

---

## 3.1 系统概念

传统视觉小说的分支选择往往「隐形」——玩家不知道早期选择如何影响后期。回响系统通过**视觉提示**让蝴蝶效应「可见」：

- 当某个早期选择影响到当前场景时，屏幕边缘闪现一个**金色涟漪**
- 涟漪中央显示一个小图标，代表那个早期选择（如：玻璃珠、顾老三的脸、林若兰的疤痕）
- 点击图标，弹出一句「回响台词」——角色或旁白的一句话，暗示因果联系

## 3.2 回响示例

```
【第四章，万国博览会地下层】
沈默潜入地下层，发现主发射器。

【屏幕边缘闪现金色涟漪】
┌────────────┐
│ 🟡         │
│  [玻璃珠]  │ ← 第一章选择「保留玻璃珠」的回响
│  「那颗珠子...原来是为了这个」│
└────────────┘
```

**具体回响对应**：

| 早期选择 | 回响位置 | 回响台词 |
|---------|---------|---------|
| 第一章保留玻璃珠 | 第四章地下层 | 「那颗珠子……原来是为了这个。」 |
| 第二章信任小翠 | 第四章潜入时 | 「她说过的话，现在开始兑现了。」 |
| 第三章救顾老三 | 第四章潜入时 | 「老三的黄包车，应该停在门口。」 |
| 第三章相信林若兰 | 第五章隔离室 | 「她给的设计图……就是这个时刻。」 |
| 第二章追问周老板 | 隐藏结局 | 「二十年前的事……终于连上了。」 |

## 6.1 Web Audio API 应用

利用 Web Audio API 实现「频率可视化」——这是纯Web方案的独特优势：

**频率匹配解谜时的实时音频反馈**：
```javascript
// 创建振荡器，频率随滑块变化
const oscillator = audioContext.createOscillator();
const gainNode = audioContext.createGain();
oscillator.connect(gainNode);
gainNode.connect(audioContext.destination);

// 滑块拖动时更新频率
slider.addEventListener('input', (e) => {
  const freq = parseInt(e.target.value);
  oscillator.frequency.value = freq;
  
  // 接近7830时音量增大，偏离时音量减小+杂音
  if (Math.abs(freq - 7830) < 50) {
    gainNode.gain.value = 1.0;
    // 播放「和谐」音效
  } else {
    gainNode.gain.value = 0.3;
    // 播放「噪音」音效
  }
});
```

**7830 Hz的超声波「不可听」效果**：
- 实际频率超出人耳范围，但 Web Audio 可以生成
- 视觉上用波形动画表示「人耳听不到，但机器在响」
- 屏幕震动（CSS `transform: translate()` 微振动）模拟体感

## 8.1 核心引擎类（Vanilla JS）

```javascript
// ===== 核心引擎类 =====
class VisualNovelEngine {
  constructor() {
    this.chapter = 1;
    this.scene = 1;
    this.params = {
      INTEL: 0, DRAMA: 0, ACTION: 0, TECH: 0,
      CLUE: 0, ZHOU: 0
    };
    this.affection = {
      lin_ruolan: 30, chen_zixuan: 30,
      gu_laosan: 30, xiao_cui: 30, bai_jinshu: 30
    };
    this.flags = {};
    this.history = []; // 选择历史，用于回响系统
  }
  
  // 显示场景
  async showScene(sceneData) {
    // 1. 加载背景图
    await this.loadBackground(sceneData.bg);
    
    // 2. 检查回响
    this.checkEchoes();
    
    // 3. 显示角色立绘
    if (sceneData.character) {
      this.showCharacter(sceneData.character, sceneData.expression);
    }
    
    // 4. 逐字显示对话
    await this.typewriter(sceneData.dialogue);
    
    // 5. 显示选择（如果有）
    if (sceneData.choices) {
      this.showChoices(sceneData.choices);
    }
  }
  
  // 选择处理
  makeChoice(choice) {
    // 更新参数
    for (let [key, val] of Object.entries(choice.effects)) {
      this.params[key] = Math.min(100, this.params[key] + val);
    }
    
    // 记录历史
    this.history.push({
      chapter: this.chapter,
      choice: choice.id,
      timestamp: Date.now()
    });
    
    // 触发回响检查
    this.checkEchoes();
    
    // 跳转到下一场景
    this.showScene(choice.nextScene);
  }
  
  // 回响系统
  checkEchoes() {
    const echoes = this.getActiveEchoes();
    echoes.forEach(echo => {
      this.showEchoRipple(echo);
    });
  }
  
  // 存档
  save(slot) {
    const data = {
      chapter: this.chapter,
      scene: this.scene,
      params: this.params,
      affection: this.affection,
      flags: this.flags,
      history: this.history,
      timestamp: Date.now()
    };
    localStorage.setItem(`vn_save_${slot}`, JSON.stringify(data));
  }
  
  // 读档
  load(slot) {
    const data = JSON.parse(localStorage.getItem(`vn_save_${slot}`));
    this.chapter = data.chapter;
    this.scene = data.scene;
    this.params = data.params;
    this.affection = data.affection;
    this.flags = data.flags;
    this.history = data.history;
  }
}
```

## 9.1 市面上视觉小说的常见模式 vs 《第七频率》

| 常见模式 | 《第七频率》的不同 |
|---------|-----------------|
| 日系二次元画风 | Art Deco故障风：民国老照片+数字glitch |
| 纯文字+立绘 | 3个解谜环节，玩家动手操作频率设备 |
| 选择只影响结局 | 回响系统：选择实时可视化反馈 |
| 单一路线游玩 | 25个主抉择×3选项=海量组合，多周目价值高 |
| 存档读档翻历史 | 蝴蝶效应图谱：通关后展示选择影响力网 |
| 桌面端为主 | 零依赖HTML5，手机/平板/PC全平台 |
| 静态BGM | Web Audio实时生成7830Hz频率音效 |

## 9.2 核心卖点

1. **「民国×科幻」独特题材**：不是穿越，是1931年的真实上海+边缘科学幻想
2. **「频率」主题贯穿玩法**：解谜、音频、视觉都围绕7830 Hz展开
3. **「Art Deco故障风」视觉**：市面上没有同类产品的美学风格
4. **「回响系统」创新**：让玩家「看见」自己的选择在发光
5. **纯Web零门槛**：无需下载安装，链接即玩
